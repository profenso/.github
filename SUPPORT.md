# Support

## Documentation

Auto-generated code documentation is published to **[wiki.profenso.com](https://wiki.profenso.com/)**. Repositories bootstrapped with the latest scaffold tooling automatically generate and publish documentation there. Check the wiki first — your question may already be covered.

## Getting help

- **Documentation** — browse [wiki.profenso.com](https://wiki.profenso.com/) for auto-generated code and project documentation
- **Bugs or feature requests** — open an issue using the relevant [issue template](https://github.com/profenso/.github/issues/new/choose)
- **Security vulnerabilities** — report privately via the repository's **Security** tab (see [SECURITY.md](SECURITY.md))
- **General questions** — ask in the team Slack channel
- **Infrastructure or access requests** — use the [Infrastructure Request](https://github.com/profenso/.github/issues/new?template=infrastructure_request.yml) or [Access Request](https://github.com/profenso/.github/issues/new?template=access_request.yml) issue templates

## Before opening an issue

1. Check existing issues — your question may already be answered
2. Pick the right template — this helps us triage faster
3. Include enough context — steps to reproduce, error messages, environment details
